# Internationalization

LiF is built as an English-first Django app with German as the first additional language.

## Current Coverage

The language foundation is active:

- Django `LocaleMiddleware` is enabled.
- Users can switch language from the sidebar footer.
- English (`en`) and German (`de`) are configured.
- The shared app shell and primary navigation have German translations.

Most page-specific forms, tables, help text, and model labels are still English. Translate those page by page so each change can be reviewed in context.

## Marking Strings

Templates should load i18n and wrap visible static text:

```django
{% load i18n %}
{% trans "Accounts" %}
```

Use `blocktrans` when a sentence includes variables:

```django
{% blocktrans with mode=current_household.get_data_mode_display %}{{ mode }} data{% endblocktrans %}
```

Python code should use lazy translations for model labels, choices, forms, and settings:

```python
from django.utils.translation import gettext_lazy as _

label = _("Accounts")
```

## Updating German Translations

Install gettext tooling if `makemessages` or `compilemessages` cannot find `xgettext` or `msgfmt`.

On macOS with Homebrew:

```bash
brew install gettext
brew link --force gettext
```

Then update and compile the German catalog:

```bash
pipenv run django-admin makemessages -l de --ignore "staticfiles/*"
pipenv run django-admin compilemessages -l de
```

Commit both files:

```text
locale/de/LC_MESSAGES/django.po
locale/de/LC_MESSAGES/django.mo
```

The `.po` file is the human-editable translation source. The `.mo` file is the compiled runtime catalog Django loads.

## Testing

Run the normal checks after changing translations:

```bash
pipenv run python manage.py test
pipenv run python manage.py check
git diff --check
```

For a quick manual smoke test, switch the sidebar language selector to German and confirm the main navigation shows labels such as `Konten`, `Einstellungen`, and `Sprache`.
